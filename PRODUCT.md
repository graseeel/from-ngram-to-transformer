# Product

## Register

product

## Users

Recruiters, engineering managers, and developers reviewing a portfolio project. They use the demo to see whether the repository is executable, understandable, and technically credible rather than just visually packaged.

## Product Purpose

From N-gram to Transformer shows the practical path from a character N-gram language model to a compact decoder-only Transformer. Success means a reviewer can run the demo, compare model readiness and outputs, inspect sampling controls, and see that Supabase-backed history is wired without the ML code depending on infrastructure details.

## Brand Personality

Precise, credible, and calm. The interface should feel like a small machine-learning lab bench: technical enough to earn trust, restrained enough to keep the models and outputs in focus, and polished enough for a portfolio review.

## Anti-references

Do not make this look like a generic AI SaaS landing page, a neon hacker toy, or a decorative model playground where visual effects compete with the generated text. Avoid oversized marketing claims, emoji-led UI, glassmorphism, gradient text, and unclear form controls.

## Design Principles

- Show working software before explanation.
- Make model state and save state explicit.
- Keep every control close to the output it affects.
- Prefer familiar product UI patterns over novelty.
- Let technical credibility come from clear structure and verified behavior.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Preserve visible focus states, readable labels, reduced-motion alternatives, and usable layouts from small mobile widths through desktop review sessions.
